Setting up VS Code Tunnels for Remote Access

VS Code tunnels allow you to securely access your locally running development server from any device with an internet connection. This is particularly useful for testing your P5.js projects on mobile devices or sharing your work with others.

Summary

What You'll Set Up

  • Secure remote access to local development server
  • Public URLs for your P5.js projects
  • Cross-device testing capabilities

Requirements

  • VS Code with tunnel support
  • GitHub account for authentication
  • Live Server running locally
  • Active development project

Access Methods

  • Direct URL links
  • QR codes for mobile devices
  • Authentication-based access
  • Secure tunnel connections

Use Cases

  • Mobile responsive testing
  • Sharing work with others
  • Cross-device compatibility
  • Remote collaboration

Part 1 - Setting up the Tunnel

Step 1 - Access the Tunnel Menu

Open VS Code and access the tunnel options from the bottom left corner.

  • Click the tunnel icon in the bottom left corner
  • Select your tunnel options

Step 2 - Turn On Tunnel for Current Session

Enable tunneling for your current VS Code session.

  • Choose "Turn on Tunnel for Current Session"
  • This will start the tunneling process

Step 3 - Sign In to GitHub

You'll need to authenticate with your GitHub account to create the tunnel.

  • Sign in with your GitHub account when prompted
  • This enables secure tunnel creation

Part 2 - Configuring Port Forwarding

Step 1 - Ensure Live Server is Running

Make sure your P5.js project is running with Live Server.

  • Your project should be running on a local port (typically 5500)
  • Live Server extension should be active

Step 2 - Open the Ports Tab

Access the ports panel to manage forwarded ports.

  • Click on the "Ports" tab in the terminal panel
  • This shows all active ports

Step 3 - Forward the Port

Forward your local development port through the tunnel.

  • Find your Live Server port (usually 5500)
  • Right-click and select "Forward Port"

Step 4 - Allow Access

Grant permission for the port forwarding.

  • Click "Allow" when prompted
  • This enables external access to your port

Step 5 - Confirm Account Access

Confirm that you want to allow tunnel access.

  • Confirm your GitHub account permissions
  • This finalizes the tunnel setup

Part 3 - Accessing Your Tunnelled Site

Step 1 - Get the Public URL

Your local server now has a public URL accessible from anywhere.

  • Copy the public URL shown in the ports panel
  • This URL can be accessed from any device

Step 2 - Handle Future Prompts

You can choose to not show certain prompts in the future.

  • Check "Don't show again" for future sessions
  • This streamlines the process

Step 3 - Open the Link

Access your tunnelled site from any browser.

  • Click the globe icon or copy the URL
  • Open in any web browser

Step 4 - Alternative Access Methods

You can also access via authentication link or QR code.

  • Use the authentication link for direct access
  • Scan the QR code with your mobile device
  • Perfect for testing on phones and tablets

Important Notes

  • Tunnels are temporary and expire when you close VS Code
  • Keep your GitHub account secure as it controls tunnel access
  • Use tunnels responsibly and only share URLs with trusted individuals
  • This is perfect for testing responsive designs and mobile interactions

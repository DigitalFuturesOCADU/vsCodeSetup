Step 1 - Open VS Code Terminal

What is the Terminal?

The Terminal (also called Command Line) is a text-based interface where you can run commands. VS Code has a built-in terminal that makes it easy to run commands without leaving your editor.

How to Open the Terminal in VS Code

There are three ways to open the Terminal in VS Code:

Method 1: Keyboard Shortcut

  • Mac: Control + ` (backtick key)
  • Windows: Ctrl + ` (backtick key)

The backtick key (`) is usually above Tab, left of the number 1

Method 2: Menu

  • Click Terminal in the top menu
  • Select New Terminal

Method 3: Command Palette

  • Press Cmd+Shift+P (Mac) or Ctrl+Shift+P (Windows)
  • Type "terminal"
  • Select Terminal: Create New Terminal

Success! When the terminal opens, you'll see a panel at the bottom of VS Code with a command prompt ready for input.

Step 2 - Ensure Node.js is Installed

Check if Node.js is Already Installed

Before installing, let's check if Node.js is already on your system. In your VS Code terminal, type:

node --version

If you see a version number (like v20.11.0), Node.js is already installed! You can skip to Step 3.

If Node.js is NOT Installed

If you see an error like "command not found" or "not recognized", you need to install Node.js:

  1. Visit nodejs.org
  2. Download the LTS version (Long Term Support) - the green button
  3. Run the installer and use all default settings
  4. Important: Close and reopen VS Code after installation
  5. Open a new terminal and verify: node --version

Why Node.js? Node.js includes npm (Node Package Manager), which lets us run the verification tool without installing it permanently.

Step 3 - Run the Setup Verification

The Simple One-Line Command

Now for the magic! In your VS Code terminal, run this single command:

npx ocadu-vscode-setup

What is npx? npx comes with npm and lets you run packages without installing them. The verification script runs directly from the internet!

What Happens Next

When you run the command:

  1. First time only: npm will download the verification package (takes 2-3 seconds)
  2. The script will automatically check your entire setup
  3. You'll see a color-coded report in the terminal
  4. The whole process takes about 10-15 seconds

Success! If you see version numbers for both commands, Node.js is properly installed.

No installation required! Every time you run npx ocadu-vscode-setup, you get the latest version automatically.

Step 4 - Understanding the Results

What Does the Script Check?

The verification script automatically checks:

  • Node.js & npm: Confirms they're installed and working

  • Repository Status (optional): Git repository, remote configuration, uncommitted changes
  • Manual Checklist: Items you need to verify yourself

Step 5 - Fix Issues (If Any)

How to Fix Failed Checks

Every failure includes How to fix instructions. Follow these steps:

  1. Focus on CRITICAL items first (üî¥) - These must be fixed
  2. Read the fix instructions - They appear right below each failure
  3. Copy and run the commands - Many fixes are one-line terminal commands
  4. Follow guide links if needed - Detailed help is available
  5. Run the script again - Verify your fixes: npx ocadu-vscode-setup

Example: Installing a Missing Extension

If the script says "P5 Project Creator is NOT installed", you'll see:

Quick install: Run this command in terminal:
  code --install-extension ultamatum.p5-project-creator

Or install manually:
  1. Open VS Code
  2. Press Cmd+Shift+X (Mac) or Ctrl+Shift+X (Windows)
  3. Search for "P5 Project Creator"
  4. Click Install

Just copy the code --install-extension command, paste it in your VS Code terminal, and press Enter!

Remember: If the final summary says "Your system is working correctly!", you're good to go even if some optional items failed!

Troubleshooting Common Issues

Command Not Found: npx

If you see "command not found: npx":

  1. Node.js is not installed or not in your PATH
  2. Install Node.js from nodejs.org
  3. Important: Close and reopen VS Code after installing
  4. Open a new terminal and try again

"VS Code is NOT installed" (but it IS installed!)

If the script can't find VS Code but you know it's installed:

On Mac:

  1. Open VS Code
  2. Press Cmd+Shift+P to open Command Palette
  3. Type "shell command"
  4. Select "Shell Command: Install 'code' command in PATH"
  5. Close and reopen your terminal
  6. Run the verification script again

On Windows:

  1. During VS Code installation, make sure "Add to PATH" is checked
  2. If already installed, you may need to reinstall with this option
  3. Download from code.visualstudio.com

Note: The script can find VS Code even if it's not in PATH, so this is usually an IMPORTANT (not CRITICAL) issue.

"Git is NOT installed"

If Git is not installed:

On Mac:

xcode-select --install

This installs Xcode Command Line Tools which includes Git.

On Windows:

  1. Download from git-scm.com/downloads
  2. Run the installer with default settings
  3. Close and reopen VS Code

"Current directory is NOT a Git repository"

This warning appears if you're not inside a Git repository folder. This is OPTIONAL - you can run the script from anywhere!

If you want repository checks:

  • Navigate to your project folder first
  • Or create a repository following the Repository Setup Guide

Missing Extensions

If extensions aren't detected:

  1. Open VS Code
  2. Click the Extensions icon in the sidebar (or press Cmd+Shift+X / Ctrl+Shift+X)
  3. Search for and install each missing extension
  4. Run the verification script again

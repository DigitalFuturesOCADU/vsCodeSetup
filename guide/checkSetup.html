Part 1 - Installing Node.js

What is Node.js?

Node.js is a JavaScript runtime that allows you to run JavaScript code outside of a web browser. We'll use it to run a verification script that checks if your development environment is properly configured.

Step 1 - Download Node.js

Visit the official Node.js website and download the LTS (Long Term Support) version for your operating system.

Download Link:
  • Node.js Official Website
‚ÑπÔ∏è Tip: Choose the LTS version (currently v20.x or v22.x). This is the recommended version for most users.

Step 2 - Install Node.js

For macOS:

  1. Open the downloaded .pkg file
  2. Follow the installation wizard
  3. Accept the license agreement
  4. Use default installation settings
  5. Enter your password when prompted

For Windows:

  1. Open the downloaded .msi file
  2. Follow the installation wizard
  3. Accept the license agreement
  4. Use default installation settings (includes npm and PATH configuration)
  5. Click "Install" and wait for completion

Step 3 - Verify Node.js Installation

Open a new terminal window (or Command Prompt on Windows) and run:

node --version

You should see a version number like v20.11.0 or similar.

Also verify npm (Node Package Manager) is installed:

npm --version

You should see a version number like 10.2.4 or similar.

‚úì Success! If you see version numbers for both commands, Node.js is properly installed.

Part 2 - Running the Verification Script

What Does the Script Check?

The verification script automatically checks:

  • Software Installation: VS Code, Git, Node.js
  • VS Code Extensions: GitLens, p5js Snippets, P5 Project Creator, Live Server, GitHub Actions
  • Git Configuration: Username, email, OCADU email check
  • Repository Status: Git repository, remote configuration, uncommitted changes
  • Manual Checklist: Items that require manual verification

Step 1 - Navigate to Your Project Directory

Open a terminal in VS Code (Terminal ‚Üí New Terminal) or use your system terminal. Navigate to your project repository:

‚ÑπÔ∏è Note: Replace path/to/your/repository with the actual path to your project folder. If you're already in VS Code, the integrated terminal should automatically open in your project folder.

Step 2 - Run the Verification Script

Run the verification script using npx (no download required):

For help and documentation, run:

‚ÑπÔ∏è What is npx? npx runs Node.js packages directly from npm without installing them. This means you always get the latest version of the verification script.
üí° Tip: You can run this command from any directory - it will check your development environment wherever you are. For repository-specific checks, run it from inside your project folder.

Understanding Criticality Levels

Each failed check includes a criticality level to help you prioritize fixes:

  • üî¥ CRITICAL: Must be fixed for development to work properly
  • üü° IMPORTANT: Recommended for full functionality, but may not be essential
  • üü¢ OPTIONAL: Nice to have, but not required for development
üí° Important: If you only have IMPORTANT or OPTIONAL failures, your system is likely working correctly! For example, VS Code not being in PATH is flagged as IMPORTANT, but if you can open VS Code normally, your setup is fine.

The script checks each section:

  1. Node.js Installation - Confirms Node.js and npm are installed
  2. VS Code Installation - Checks if VS Code is accessible from the command line
  3. VS Code Extensions - Verifies all 5 required extensions are installed
  4. Git Installation & Configuration - Checks Git setup and user configuration
  5. Local Repository - Validates your repository structure and status
  6. Manual Verification Checklist - Lists items you need to verify manually
  7. Summary Report - Shows overall completion percentage

  3. Run the provided commands - Many fixes are just one terminal command
  4. Follow the guide links - Detailed instructions in the referenced guides

Troubleshooting Common Issues

"VS Code not found" Error

If the script can't find VS Code:

  • macOS: Open VS Code, press Cmd+Shift+P, type "shell command", and select "Install 'code' command in PATH"
  • Windows: Make sure you checked "Add to PATH" during VS Code installation. You may need to reinstall VS Code.

"Git not found" Error

If Git is not installed:

  • macOS: Git should come with Xcode Command Line Tools. Run xcode-select --install in terminal
  • Windows: Download and install Git from git-scm.com/downloads

"Not a Git repository" Warning

This means you're not running the script from inside a Git repository. Either:

  • Navigate to your project repository folder first
  • Or create a new repository following the Repository Setup Guide

Extensions Not Found

If extensions aren't detected:

  1. Open VS Code
  2. Click the Extensions icon in the sidebar (or press Cmd+Shift+X / Ctrl+Shift+X)
  3. Search for and install each missing extension
  4. Run the verification script again
